Файл набора данных позволяет в автоматическом режиме подставлять введённые значения в шаблон рекламного баннера. Ниже рассмотрено применение на примере клиента Joom (настоящий скрипт является универсальным инструментом).

В первой строке набора данных содержатся имена переменных (её необходимо оставить без изменения или обговорить идеи с дизайнером). Каждая последующая строка соответствует отдельному баннеру. Очень важно передавать значения переменным в порядке, соответствующем порядку следования их названий в первой строке, сохраняя пунктуацию и раскладку клавиатуры (латиница). В случае использования рациональных чисел для обозначения цены товара в качестве разделителя необходимо использовать символ "." (например, 47.54 ₽).

Ниже приведён пример файла набора данных: 
pic, before, after, btn
1-r.jpeg, 2000 ₽, 1000 ₽, true
2-u.jpeg, 3000 ₴, 1500 ₴, false
Разбор содержимого файла:
Первая строка содержит имена переменных: 
pic, before, after, btn
Вторая и третья строка содержат значения для вышеуказанных переменных, разделённые запятыми, а именно:
1-r.jpeg - имя изображения товара в текущей директории. Файлы должны именоваться в соответствии с примером, где "r", "u", "b", "k" - категория баннера (r - russia, u - ukraine, b - belarus, k - kazakhstan), "1" - порядковый номер баннера (не должен совпадать в рамках одной категории). В качестве разделителя использовать символ "-";
2000 ₽ - стоимость товара без учёта скидки;
1000 ₽ - стоимость товара с учётом скидки;
true - указывает на наличие кнопки.
Готовый текстовый файл необходимо сохранять в формате .txt, избегая возможного появления дополнительных строк с метаданными текстового редактора.
